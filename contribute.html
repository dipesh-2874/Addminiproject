How to Contribute

You can contribute in two ways: add a mini project, or improve the main website with feature and enhancement work. Mini project submissions follow additional submission restrictions.

Mini Project Rules

Before starting a mini project, review the official rules. Mini project issues and PRs are validated for ownership, active submission limit, and 7-day restriction.

Contribution Steps

  1. Fork the Repository

    Click the Fork button on GitHub.

  2. Clone Your Fork

    git clone https://github.com/your-username/Addminiproject.git
cd Addminiproject

  3. Create a New Branch

    git checkout -b your-branch-name

  4. Make Your Changes

    Add your mini project inside projectforcontributor or work on approved website improvements.

  5. Stage and Commit

    git add .
git commit -m "Add: brief description of contribution"

  6. Push to GitHub

    git push origin your-branch-name

  7. Create Pull Request

    Open your fork on GitHub and click Compare & Pull Request.

Mini Project Rules Confirmation

Please confirm the required rules before opening a mini project issue.

  • Only one active project submission is allowed at a time.
  • A minimum 7-day gap is required after your previous submission.
  • You can work only on issues raised by yourself.

Backend checks enforce one active mini project, issue ownership, and 7-day restriction.
